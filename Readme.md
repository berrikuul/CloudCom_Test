### Тестовое задание для студентов

1. Почитать документацию:
   1. <https://airflow.apache.org/docs/>
   2. <https://docs.min.io/enterprise/aistor-object-store/>
2. Поднять инфраструктуру в docker (S3 хранилище, Airflow, Postgres)
3. Написать даг по перемещению данных из t1 в t2
   1. Посмотреть датасет <https://www.kaggle.com/datasets/rai220/russian-cyrillic-names-and-sex> или выбрать любой ваш
   2. Создать 2 одинаковые таблицы t1 и t2 на основе выбранного датасета, кроме пары моментов:
      1. В таблице t1 должно быть поле id bigserial primary key, в таблице t2 id должно быть просто bigint
      2. В таблице t1 должно быть поле created timestamp default now(), в t2 должно быть created timestamp и load_time timestamp default now() (показать время создание и время загрузки данных)
      3. Наполнить таблицу t1 данными из датасета, первые 100 строк (весь датасет грузить не надо)
   3.  Написать даг, который запускается каждый день и перегружает данные из таблицы t1 в таблицу t2. Перед тем, как переместить данные в таблицу t2 ее необходимо очистить. Все перемещение должно происходить в транзакции
4. Написать даг, который загружает данные из t1 в s3 хранилище по партициям
   1. Даг должен проверить, есть ли данные за сегодняшний день (created between data_interval_start и data_interval_end)
   2. Если таких данных нет, то даг должен пропустить день (state of task instance must be `skipped`, должен быть розовый квадратик) 
   3. Если данные есть, необходимо сформировать дневную папку (название должно формироваться из data_interval_start в iso format, например 2025-01-01, 2025-01-02 и тд)
   4. В этой папке формировать JSON файл, в который необходимо выгрузить все данные из таблицы t1 за сегодняшний день (created between data_interval_start и data_interval_end).
